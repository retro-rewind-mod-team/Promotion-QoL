# Retro Rewind – Promotion QoL

Automatically distributes flyers to all walkby NPCs outside the store,
so you don't have to hand them out manually one by one.

---

## Requirements

- **UE4SS** must be installed first.
  Follow the installation instructions on the [UE4SS Nexus page](NEXUS_UE4SS_LINK) before proceeding.

---

## Installation

1. Extract the zip file into your game folder:
   ```
   Steam\steamapps\common\RetroRewind\RetroRewind\Binaries\Win64\ue4ss\Mods\
   ```

---

## How It Works

When you flip the Open/Closed sign, the mod finds all walkby NPCs currently
outside and triggers the flyer interaction for each one. NPCs that spawn
while the store is open are queued and receive their flyer after a short
ingame delay.

Each NPC reacts naturally — some will say "Oh, I love movies!" and head
toward the store, others will refuse. The conversion chance is read directly
from the game's AI Director and reflects your store's popularity and
decoration level. A fully upgraded store has a higher chance than a new one.

> **Note:** It is unclear whether the game's AI Director compensates for the
> additional customers generated by flyers. You may not see a significant
> spike in customer numbers compared to distributing flyers manually.
> If you want to experiment, you can set a custom `conversionRate` in the
> config (see below).

---

## Configuration

Open `main.lua` in a text editor and edit the `CONFIG` section at the top:

```lua
local CONFIG = {
    flyerDelay     = 14,   -- ingame minutes before giving flyer to new NPC
    conversionRate = nil,  -- nil = use native store popularity rate
                           -- 0.0 to 1.0 = override (1.0 = all NPCs accept)
}
```

| Option           | Description                                                        |
|------------------|--------------------------------------------------------------------|
| `flyerDelay`     | How many ingame minutes to wait before giving a flyer to a new NPC |
| `conversionRate` | `nil` = native rate, `0.0`–`1.0` = custom rate, `1.0` = all accept |

## Technical Notes (for curious modders)

- `OpenSign_C:Change Sign` — detects store opening
- `WeatherSystem_C:Timer Event - Add one minute` — processes NPC queue
- `AI_Director_C:Return Weight Chance of Spawning AI by Store Popularity and Deco` — native conversion chance
- `AI_WalkBy_Character_C:Walker Accept the flyer` / `Walker refuse the flyer` — triggers native NPC reaction
- `AI_WalkBy_Character_C:Transform the Walker into a Client` — counted to track customer conversions
- `NotifyOnNewObject` on `AI_WalkBy_Character_C` — queues newly spawned NPCs

The conversion chance scales with store popularity: a small new store returns
~0.11, a fully upgraded store returns ~0.27. This reflects the game's
intended balance.
